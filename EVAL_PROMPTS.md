# 智能背单词应用 — AI 评测 Prompt 题库

> 本项目基于 React (Vite) + Node.js + SQLite 的智能背单词应用。以下题目均可在前端页面完成验证，设计为需要多轮对话协作才能完整落地的任务。

---

## Feature 迭代（在已有功能基础上的平滑扩展）

### F-01 [Feature迭代]

主页右侧「学习进度」折线图目前是按已学单词序号生成的伪日期数据（Day 1、Day 2…），并不能反映真实的学习节奏。请把这块改造成按自然日聚合的统计：后端需要记录每次「标为已掌握」的时间戳并按 `YYYY-MM-DD` 分组返回，前端折线图横轴显示最近 14 天的日期、纵轴为当天新掌握单词数；如果某天没有学习记录则显示 0 而不是跳过该点。图表 Tooltip 要展示具体日期和数量，并在图表下方保留「今日已学」计数但与折线图数据口径一致。整体 UI 仍保持现有的玻璃拟态深色风格，改动完成后我能在 Dashboard 连续几天学习后看到真实日期曲线。

### F-02 [Feature迭代]

词汇量测试页目前每道题用户可以无限思考，缺少时间压力感。请在测试流程中为每一题增加倒计时：默认每题 20 秒，顶部显示环形或条形进度条，剩余 5 秒时进度条变红并轻微脉冲动画；超时未作答则自动记为「不认识」并进入下一题。测试设置入口放在开始测试前的确认页，允许用户在 15/20/30 秒三档间选择。倒计时在切换题目时重置，暂停功能不需要做。完成后结果页要额外展示「超时题数」统计。所有交互需在 `/test` 路由页面可直接操作验证。

### F-03 [Feature迭代]

「复习已掌握单词」弹窗目前只能按时间倒序浏览全部记录，单词多了以后很难找。请给复习弹窗增加搜索框（支持按英文单词或中文释义模糊匹配）、按掌握时间排序切换（最新优先 / 最早优先）、以及按单词 rank 难度区间筛选（基础 / 初级 / 中级 / 中高级 / 高级 / 专业，对应现有 `difficulty_level` 字段）。筛选条件变化时列表即时更新，无匹配结果时显示空状态提示。搜索和筛选状态在关闭弹窗再打开后应重置。样式与现有弹窗一致，不新增独立路由。

### F-04 [Feature迭代]

每日推荐单词卡片目前只有「标为已掌握」和「跳过」两个操作，用户无法标记「以后还想看」。请在推荐卡片区域新增「收藏」按钮（心形图标），点击后该词写入用户的收藏列表且当天不再作为推荐出现，但未计入已掌握。Dashboard 快捷操作区新增「我的收藏」入口，打开侧边抽屉展示收藏单词，支持取消收藏（取消后该词可再次被推荐）和点击播放发音。后端需要新建收藏相关表和 API，前端收藏状态在刷新页面后仍保持。

### F-05 [Feature迭代]

用户学习某个单词时往往想记自己的联想笔记，但当前单词卡片不支持个人备注。请在推荐卡片的例句区域下方增加可折叠的「我的笔记」区块：默认展示已有笔记或「点击添加笔记」占位，点击进入编辑模式（多行文本框），保存后调用后端接口持久化到用户-单词关联表；再次打开该词（若被重新推荐或从复习列表进入）时能加载历史笔记。笔记仅自己可见，单条上限 500 字，超出时前端提示。空笔记保存等同于删除。请保证笔记与具体单词和用户绑定，换账号看不到彼此的笔记。

### F-06 [Feature迭代]

跳过单词后现有逻辑是一小时内不再推荐，但用户无法查看自己跳过过什么、也无法反悔。请新增「跳过记录」管理能力：Dashboard 快捷操作增加「跳过记录」入口，弹窗列出近 30 天内跳过的单词（含跳过时间），每条提供「重新加入推荐池」按钮，点击后清除跳过限制使该词可以再次被推荐。列表支持分页或虚拟滚动以应对较多记录。后端需扩展 `learning_history` 的 skipped 状态查询接口，并支持删除或更新跳过记录。重新加入推荐池后不需要自动跳到该词，用户手动刷新推荐即可验证效果。

### F-07 [Feature迭代]

当前登录后主页只显示注册时的用户名，用户无法修改展示信息。请在 Dashboard 右上角用户区域增加头像和设置入口，打开设置面板后可修改「昵称」（默认同用户名，最长 20 字）和从预设 12 个渐变色头像中选择头像样式（不需要上传图片）。保存后主页问候语显示昵称而非用户名，头像以圆形徽章展示。后端 `users` 表扩展 `nickname` 和 `avatar_color` 字段，并提供 `PATCH /api/me` 更新接口；修改后刷新页面数据不丢失。用户名仍用于登录且不可在此修改。

### F-08 [Feature迭代]

现在 JWT 令牌 24 小时过期且前端没有「记住我」选项，用户每次关闭浏览器都要重新登录。请在登录页增加「记住我」复选框：勾选时 token 存入 `localStorage` 且后端签发 30 天有效期的 token，未勾选则维持现有 `sessionStorage` + 24 小时策略。应用启动时 `AuthContext` 需根据存储位置正确恢复会话，过期 token 自动清除并跳转登录页且给出中文提示。登出时两种存储中的 token 都要清除。注册页不需要此选项。完成后我可以通过勾选记住我、关闭再打开浏览器仍保持登录状态来验证。

### F-09 [Feature迭代]

词汇量测试完成后的结果页目前只展示总词汇量估算、正确率和题数，信息较单薄。请在结果页增加按 `difficulty_level` 分组的答题明细：每个难度等级显示答题数、正确数、正确率进度条，并用颜色区分表现好/一般/差。若某等级没有抽到题则不显示该组。底部增加「薄弱等级」文字建议（正确率最低的等级）。数据由前端根据测试过程中的答题记录计算即可，不要求改后端提交接口。用户重测后结果页展示最新一次的数据。所有内容在 `/test` 完成流程后即可看到。

### F-10 [Feature迭代]

应用目前没有连续学习天数的概念，用户难以感知坚持情况。请实现「学习连续天数」功能：后端根据 `learning_history` 中 status 为 learned 的记录按自然日去重，计算当前连续有学习记录的天数（今天没学则连续天数截止到昨天）；Dashboard 头部在词汇量下方展示火焰图标和「已连续学习 X 天」。若今天尚未学习，显示「今日还未学习，连续记录面临中断」的弱提示。连续天数达到 3/7/14/30 天时在主页弹出一次祝贺 toast（同一里程碑只弹一次，存 localStorage）。请确保跨时区按服务器时间或统一 UTC 处理并注明你的选择。

### F-11 [Feature迭代]

推荐单词卡片目前是平铺展示释义和例句，缺少背单词常见的翻转卡片体验。请在单词卡片上增加「卡片模式」切换开关：开启后默认只展示英文单词和音标，点击卡片或按空格键翻转显示释义和例句，再次翻转隐藏；翻转使用 Framer Motion 3D 翻转动画。标为已掌握或跳过后自动翻回正面。切换回「经典模式」时恢复现有布局。模式偏好存入 localStorage。两种模式下操作按钮始终可见且可用。用户可在 Dashboard 直接切换并体验完整学习流程。

### F-12 [Feature迭代]

i+1 推荐算法目前固定为略高于用户词汇量，部分用户觉得太简单或太难。请在 Dashboard 推荐区域上方增加「推荐难度」滑块，范围对应「巩固复习（i-500）」「标准 i+1（i~i+1500）」「挑战模式（i+1500~i+3000）」三档，拖动后实时更新推荐单词，并调用后端时在请求中传递偏好参数，后端调整现有推荐 SQL 的 rank 区间权重。切换难度后若当前展示的单词不符合新区间则自动拉取新推荐。默认档为标准 i+1，偏好存入用户表并在下次登录恢复。页面上用简短中文说明每档含义。

### F-13 [Feature迭代]

「复习已掌握单词」目前只是静态列表，没有间隔重复复习机制。请基于艾宾浩斯简化模型为已掌握单词计算「下次复习时间」（掌握后第 1/3/7/14 天），Dashboard 快捷操作显示「待复习 N 个」角标；点击「复习已掌握单词」时默认只展示到期单词，并提供「查看全部」切换。每个单词显示距下次复习还有几天或「已到期」。在复习弹窗中增加「确认复习」按钮，点击后该词复习次数加一并重新推算下次复习时间。需要新建复习计划表和后端 API，前端到期数量要实时准确。

### F-14 [Feature迭代]

用户有时想把已学单词导出到别的工具，但目前没有导出能力。请在复习已掌握单词弹窗底部增加「导出为 CSV」按钮，点击后将当前列表（含英文、音标、词性、释义、掌握时间）生成 CSV 文件并触发浏览器下载，文件名格式 `vocabulary-用户名-日期.csv`，UTF-8 带 BOM 以便 Excel 正确显示中文。若列表为空按钮禁用。若用户开启了搜索或筛选，则只导出当前筛选结果。不需要后端参与，纯前端实现，但数据需来自已有 stats 接口返回的完整字段。

### F-15 [Feature迭代]

词汇量测试一旦开始就无法中途离开，用户误触返回会丢失进度。请为测试流程增加「进度保存」：测试进行中把当前题号、ability 值、已答记录序列化存入 `sessionStorage`；用户中途离开再进入 `/test` 时检测到有未完成进度则弹窗询问「继续上次测试」或「重新开始」，继续则从断点恢复。完成或主动放弃后清除保存的进度。顶部增加「放弃测试」文字按钮，点击需二次确认。进度恢复后题目顺序和已答统计与离开前一致。不要在后端持久化半成品测试。

### F-16 [Feature迭代]

Dashboard 目前只展示已学总数，缺少每日目标激励。请增加「每日学习目标」功能：用户可在侧边栏设置每日掌握单词数目标（默认 5，可选 3/5/10/15/20），设置项下方展示圆环进度条显示今日已完成占目标的百分比，达标后圆环变金色并显示「今日目标已达成」。今日完成数按自然日统计当天 status 为 learned 的记录数，与主页折线图数据口径一致。目标值存入后端用户偏好字段。未达标时主页推荐卡片上方显示温和提示文案。所有 UI 集成在 Dashboard 右侧栏，不新增页面。

### F-17 [Feature迭代]

推荐卡片的例句目前整句同色显示，目标单词不够突出。请在例句渲染时自动识别并高亮其中的目标英文单词（大小写不敏感，处理常见词形变化如复数加 s、过去式加 ed 的基本情况即可，不需要 NLP 库）：高亮用语义色（如 indigo-400）加粗，其余文字保持原样式。若例句中多次出现则全部高亮。若无法匹配则原样展示并在开发者控制台 warn 一条日志。高亮逻辑封装为可复用函数。标为已掌握切换到下一张卡片后高亮应正确应用于新单词例句。在 Dashboard 直接可见效果。

### F-18 [Feature迭代]

后端已有 `/api/recommend/batch` 接口可一次获取多个推荐词，但前端未使用，用户每次只能学一个。请在 Dashboard 增加「批量学习模式」开关：开启后一次展示 3 张精简单词卡片（网格或横向滑动），每张有独立的掌握和跳过按钮；任一张操作后仅刷新该槽位为新推荐，不影响其他槽位。关闭批量模式恢复现有单卡片体验。批量模式状态存入 sessionStorage。接口失败时单槽显示错误占位并可重试。布局在移动端改为纵向堆叠且仍可操作。我要能在主页同时看到 3 个推荐词并分别标记掌握。

### F-19 [Feature迭代]

用户注册后无法修改密码，只能重新注册账号。请在 Dashboard 设置面板增加「修改密码」区域：需输入当前密码、新密码、确认新密码，前端校验两次新密码一致且长度不少于 6 位，提交后调用 `POST /api/me/change-password`，后端验证当前密码正确后更新 bcrypt 哈希。成功后提示重新登录并自动登出。错误时展示中文原因（当前密码错误等）。不可与用户名修改混为一谈。完成后我能在前端完成整条改密流程并用新密码登录验证。

### F-20 [Feature迭代]

用户多次重测词汇量后无法看到历史变化趋势。请在 Dashboard 快捷操作区增加「词汇量历史」入口，打开弹窗展示折线图：横轴为每次完成测试的日期时间，纵轴为估算词汇量，数据来自后端新建的 `vocab_test_results` 表（在现有 `/api/test/submit` 时额外写入一条汇总记录，含 vocab_size、正确率、题数）。仅展示最近 10 次，支持 hover 查看详情。若不足两次测试显示「完成至少两次测试后可查看趋势」。弹窗内提供「去测试」跳转按钮。图表风格与现有 Recharts 学习进度图统一。

## 0-1（简单完整模块构建）

### O-01 [0-1]

单词浏览模块：在主页增加入口，进入独立的单词浏览页面。左侧提供按难度等级（基础至专业共六档）的多选筛选，右侧以卡片流形式浏览词库，每张卡片呈现单词、音标、词性、中文释义和常用程度标识，可按词汇难度或使用频率排序，点击卡片可展开查看例句。列表采用分页加载，避免一次拉取过多数据。用户可将感兴趣的单词加入个人学习计划，服务端记录偏好后，主页的每日推荐会优先推送这些单词，帮助用户围绕自身关注点系统学习。

### O-02 [0-1]

拼写挑战模块：在主页快捷操作区增加游戏入口，进入听音拼写挑战页面。每局包含十道题，系统朗读单词发音并给出中文释义，用户在输入框中拼写英文单词，提交后立即显示对错与正确答案，局末汇总得分与用时。开始前展示简要规则说明，挑战过程中可跳过当前题目，结束后支持一键开启新一局。题目难度围绕用户当前词汇量上下浮动选取，既保持挑战性又避免过于偏难。服务端负责出题与成绩存档，前端承担发音播放、答题交互和结果展示。

### O-03 [0-1]

学习排行榜模块：在主页增加排行榜入口，展示用户之间的学习活跃度对比。榜单列出掌握单词数排名前二十的学员，显示名次、昵称、本周或本月掌握词数以及词汇量估值，前三名以金、银、铜徽章突出展示。页面顶部可在「本周」与「本月」两个维度间切换，若当前用户未进入前二十名，列表底部单独呈现其名次与数据，方便对照差距。排行数据由服务端按时间段统计已掌握单词记录生成，前端负责榜单渲染与切换交互。

### O-04 [0-1]

主题标签学习模块：为词库中的单词打上主题标签（如商务、旅游、学术、科技、日常口语等八个方向），并提供独立的主题学习页面。页面顶部以胶囊标签切换不同主题，主体区域流式展示该主题下的单词卡片，用户可选择某一主题作为当前学习方向。选定主题后，主页推荐仅推送该主题范围内的单词；用户也可随时取消主题限定，恢复全局推荐。标签与单词的关联关系由服务端维护并初始化，前端负责主题浏览、偏好设定，并与主页推荐逻辑联动。

### O-05 [0-1]

消息中心模块：在主页顶部增加铃铛图标作为消息入口，有未读消息时显示红色角标。消息中心页面按时间倒序展示个人通知，涵盖复习到期提醒、学习目标达成、排名变动和系统公告等类型。点击单条消息可展开查看详情并标记为已读，支持单条删除和一键全部已读，铃铛角标随未读数量实时更新。当用户的学习行为触发相应事件时，由服务端写入通知记录，前端负责列表展示、已读状态管理和角标同步。

### O-06 [0-1]

主题切换模块：为全站提供浅色与深色两套视觉主题，覆盖登录、注册、词汇测试和主页等所有页面。主页顶部放置太阳/月亮切换按钮，用户选择的主题偏好在本地记住，应用启动时优先应用已保存的主题，避免页面闪烁。浅色模式下玻璃拟态面板调整为浅色半透明质感与柔和阴影，文字与背景保持足够对比度。主题切换时各页面元素带有平滑过渡动画，整体风格与现有极光玻璃拟态设计语言保持一致。

### O-07 [0-1]

新手引导模块：为首次使用应用的用户设计分步引导流程。当用户尚未完成词汇量测试且没有任何学习记录时，登录后自动进入引导页而非主页。引导共分四步：欢迎页介绍产品价值与核心功能、引导完成词汇量测试（可跳过）、通过示例卡片演示如何标记单词为已掌握、完成页祝贺并进入主页。每步底部显示进度指示，支持「下一步」与「跳过引导」。引导完成后在服务端记录完成状态，后续登录不再重复展示；引导过程中可用聚光灯效果高亮关键操作区域，降低上手门槛。

### O-08 [0-1]
